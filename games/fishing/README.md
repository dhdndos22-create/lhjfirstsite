# 피싱월드 공통 패널 UI 버전

## 적용 내용

상점, 인벤토리, 장비, 뽑기, 업적, 우편이 하나의 공통 패널 화면을 사용합니다.

공통 화면에 포함되는 항목:

- 상점 배경 이미지
- 상단 레벨 / 골드 / 에너지 UI
- 반투명 남색 패널 이미지
- 화면 제목
- 선택형 카테고리 탭
- 기능별 콘텐츠 슬롯
- 선택형 페이지 이동 영역
- 오른쪽 위 공통 뒤로가기 버튼

상품카드, 인벤토리 카드, 장비 카드는 공통 UI와 분리되어 있습니다.
각 기능은 `#commonPanelBody` 슬롯에 독립 컴포넌트를 렌더링하면 됩니다.

## 주요 파일

- `js/common-panel-ui.js`
  - 공통 화면 정의
  - 카테고리 탭 생성
  - 콘텐츠 슬롯 관리
  - 페이지 이동 UI 관리
- `css/common-panel-ui.css`
  - 공통 패널 위치와 디자인
  - 뒤로가기 버튼 및 카테고리 스타일
- `js/fishing-world.js`
  - 로비 메뉴와 공통 패널 연결
  - 플레이어 상태 UI 공통 출력

## 새 메뉴 추가 예시

`js/common-panel-ui.js`의 `PANEL_DEFINITIONS`에 항목을 추가하고,
로비 메뉴 버튼의 `data-menu` 값을 같은 키로 지정하면 됩니다.

## 공통 게임 데이터와 유저 저장 데이터

이번 버전부터 게임의 고정 정보와 유저별 진행 정보를 분리합니다.

### 공통 게임 데이터

```text
js/data/
├─ game-config.js       # 최대 레벨, 초기 골드/에너지 등
├─ level-data.js        # 레벨별 필요 경험치
├─ items.js             # 소모품·미끼 정의
├─ equipment.js         # 장비 정의
├─ fish.js              # 물고기·도감 원본 정보
├─ achievements.js      # 업적 조건·보상
├─ shop-products.js     # 상점 상품과 지급 보상
└─ game-data.js         # 위 데이터를 한 곳에서 다시 export
```

공통 데이터는 모든 계정이 함께 사용하는 읽기 전용 정보입니다.

### 유저 저장 데이터

```text
js/state/
├─ player-save-template.js  # 신규 계정 기본 저장 구조
└─ player-state.js          # 현재 접속한 유저 상태와 기본 변경 함수

js/services/
└─ save-service.js          # 계정별 localStorage 저장·불러오기·정규화·구버전 이전
```

신규 계정 기본값:

```text
레벨 1
경험치 0
골드 0
에너지 10
기본 낚싯대 1개 보유 및 장착
```

저장 키는 로그인 사용자마다 분리됩니다.

```text
fishingWorldSave:<username>
```

기존 버전의 `fishingPlayer:<username>` 저장값도 최초 불러오기 때 새 구조로 자동 이전합니다.

## 스테이지 화면 기능
- 로비의 `낚시하기` 버튼을 누르면 스테이지 선택 화면으로 이동합니다.
- 좌우 화살표 또는 하단 점 버튼으로 5개 스테이지를 넘겨 볼 수 있습니다.
- 해금된 스테이지는 `입장` 버튼으로 전용 낚시 화면에 들어갑니다.
- 스테이지 배경 이미지는 현재 임시 그라데이션이며, `js/data/stages.js`의 `backgroundImage`에 추후 연결할 수 있습니다.
- 스테이지 2~5는 레벨과 이전 스테이지 도감 100% 조건을 모두 만족해야 해금됩니다.
