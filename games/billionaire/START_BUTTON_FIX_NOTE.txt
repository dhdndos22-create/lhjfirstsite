억만장자 시작 버튼 안정화 패치

수정 파일
- js/main.js
- js/database.js

수정 내용
1. 시작 버튼 이벤트를 콘텐츠 초기화보다 먼저 등록
2. 각 콘텐츠 초기화를 개별 try/catch로 분리
3. DOMContentLoaded 이후 안전하게 초기화
4. Supabase 객체가 없을 때 모듈 import 단계에서 즉시 throw하지 않음
5. 실제 DB 접근 시점에 Supabase 클라이언트를 생성
6. 시작 실패 시 실제 오류 메시지를 알림에 표시
7. 필수 DOM 요소 null 검사 추가

이제 펫/도박/직업 등 일부 콘텐츠 초기화에 문제가 발생해도 시작 버튼 이벤트 자체는 유지된다.
